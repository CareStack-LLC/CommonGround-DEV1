# CommonGround Professional Portal - Complete Report Specification

**Purpose:** Define every report type attorneys can generate, what's in each report, and when to use it.

---

## Report Categories

### 1. **Court Submission Reports** (Primary Use Case)
Reports designed to be filed with court as evidence

### 2. **Client Communication Reports**
Reports to share with clients about case status

### 3. **Internal Case Management Reports**
Reports for attorney's own case preparation

### 4. **Billing Documentation Reports** (Future)
Reports to justify fees and services

---

## 1. COURT SUBMISSION REPORTS

These are SHA-256 verified, formatted for court filing, and designed to be admissible as evidence.

---

### 1.1 Full Compliance Report

**File Name:** `Compliance_Report_[CaseName]_[DateRange].pdf`

**Purpose:** Comprehensive overview of both parents' compliance with court order or agreement. Primary evidence document for custody modification or contempt hearings.

**When to Use:**
- Preparing for custody modification hearing
- Contempt motion (showing parent's pattern of non-compliance)
- Response to modification request (showing good compliance)
- Review hearings
- Mediation preparation

- Case: Smith v. Johnson
- Case Number: FL-2024-12345
- Court: Los Angeles County Superior Court
- Date Range: January 1, 2024 - March 31, 2024
- Generated: February 26, 2024
- Generated by: John Smith, Esq. (State Bar #123456)
- SHA-256 Hash: [hash]
- CommonGround Case ID: [ID]

Trend: Declining (↓ 12% from previous quarter)

• Parent B (Mike Johnson): 58% compliance
• Primary Issues: Missed exchanges (Parent B), late support payments (Parent B)

### 1.2 Exchange Compliance Report (Focused)

**File Name:** `Exchange_Compliance_[CaseName]_[DateRange].pdf`

**Purpose:** Narrow focus on custody exchange compliance only. Used when exchange violations are the primary issue.

**When to Use:**
- Contempt motion specifically for missed exchanges
- Modification request based on exchange non-compliance
- When support and communication are compliant (no need for full report)

**Contents:**
- Cover page (same format as Full Compliance)
- Executive summary (exchanges only)
- **Detailed exchange log** (full section from Full Compliance Report)
- GPS verification screenshots
- Communication about exchanges
- Pattern analysis (days of week, parent responsible, reasons given)
- Appendices with supporting evidence

**Page Count:** 5-10 pages

---

### 1.3 Financial Compliance Report (Focused)

**File Name:** `Financial_Compliance_[CaseName]_[DateRange].pdf`

**Purpose:** Child support payment history and disputed expenses. Used for contempt, modification, or arrears collection.

**When to Use:**
- Child support contempt motion
- Arrears calculation for wage garnishment
- Modification of support amount
- Disputed expense resolution

**Contents:**
- Cover page
- Executive summary (financial only)
- **Complete payment history** (every payment with dates and amounts)
- Arrears calculation with interest (if applicable)
- **Disputed expenses detailed log** (every dispute with resolution status)
- ClearFund transaction history
- Payment method documentation
- Evidence (receipts, screenshots of payments)
- Outstanding balance summary

**Special Features:**
- Running balance calculation
- Interest calculation (if state requires)
- Payment pattern analysis (always late by X days)
- Comparison to court-ordered amount

**Page Count:** 5-12 pages

---

### 1.4 Communication Analysis Report (ARIA Focus)

**File Name:** `Communication_Analysis_[CaseName]_[DateRange].pdf`

**Purpose:** Document hostile communication patterns. Used when high-conflict communication is endangering children or violating orders.

**When to Use:**
- Request for communication restrictions
- Evidence of hostile co-parenting environment
- Request for parenting coordinator
- Contempt for violating communication protocols
- Showing pattern for custody modification

**Contents:**
- Cover page
- Executive summary (communication only)
- **Complete ARIA intervention log** (every intervention with before/after)
- Threat analysis (if any threats detected)
- Communication pattern analysis:
  - Time of day patterns
  - Trigger events
  - Escalation patterns
  - Parent comparison (who initiates hostile communication)
- Message volume analysis
- Response time analysis (cooperation indicator)
- Examples of successful co-parenting communication (if any)
- Recommendations for communication restrictions

**Special Features:**
- **Before/After comparison** for every ARIA intervention
- Sentiment analysis over time
- Word cloud of common hostile phrases
- Timeline showing escalation

**Page Count:** 10-20 pages (can be extensive in high-conflict cases)

---

### 1.5 Emergency Motion Support Report

**File Name:** `Emergency_Evidence_[CaseName]_[Incident Date].pdf`

**Purpose:** Rapid evidence compilation for emergency motions (ex parte requests, restraining orders, immediate custody changes).

**When to Use:**
- Threat detected by ARIA
- Pattern of dangerous behavior
- Child safety concerns
- Need emergency hearing within 24-48 hours

**Contents:**
- **Incident Summary** (what happened, when, evidence)
- Relevant message threads (showing threat or pattern)
- Recent compliance context (is this an isolated incident or pattern?)
- Child safety concerns
- Prior incidents (if any)
- Immediate danger assessment
- Recommended emergency orders

**Special Features:**
- **Generated in <5 minutes** for urgent filing
- Focuses on incident + immediate context
- Includes only most relevant evidence
- Can be generated at any time (not just quarter-end)

**Page Count:** 3-8 pages (intentionally brief for emergency review)

---

### 1.6 Modification Request Support Report

**File Name:** `Modification_Support_[CaseName]_[DateRange].pdf`

**Purpose:** Evidence package to support request to modify custody, support, or parenting time.

**When to Use:**
- Filing request for order modification
- Responding to other parent's modification request
- Showing substantial change in circumstances
- Preparing for mediation on modifications

**Contents:**
- Cover page
- **Substantial Change Analysis:**
  - What has changed since last order?
  - When did change occur?
  - Is change permanent or temporary?
  - How does change affect children?
- Current compliance vs. past compliance (trend)
- Evidence supporting requested modification
- Impact on children (documented)
- Proposed new arrangement
- Why proposed arrangement serves children's best interest

**Special Features:**
- **Comparison view:** Old order vs. Current compliance vs. Proposed order
- Timeline of changes
- "Best interest" factors addressed

**Page Count:** 8-15 pages

---

### 1.7 Custody Evaluation Support Report

**File Name:** `Custody_Evaluation_Data_[CaseName]_[DateRange].pdf`

**Purpose:** Objective data package for custody evaluators (psychologists, social workers, GALs).

**When to Use:**
- Court-ordered custody evaluation
- Providing data to Guardian ad Litem
- Parenting coordinator review
- Mediation with objective third party

**Contents:**
- Cover page
- Introduction (what is CommonGround, how data is collected)
- **Objective metrics only** (no attorney argument):
  - Exchange on-time percentage
  - Support payment history
  - Communication volume and patterns
  - ARIA intervention frequency
  - Agreement compliance
- Charts and graphs (visual data representation)
- Raw data appendices
- Note: "This report contains factual data only. No legal conclusions or recommendations."

**Special Features:**
- **Neutral tone** (evaluator will draw conclusions)
- Heavy use of charts/graphs
- Raw data in appendices
- Both parents' data presented equally

**Page Count:** 10-15 pages

---

### 1.8 Discovery Response Report

**File Name:** `Discovery_Response_[CaseName]_[Request Number].pdf`

**Purpose:** Respond to discovery requests (interrogatories, requests for production) with CommonGround data.

**When to Use:**
- Responding to interrogatories about compliance
- Producing documents requested by opposing counsel
- Providing evidence in response to specific discovery requests

**Contents:**
- Cover page referencing discovery request
- Point-by-point response to each request
- Relevant data from CommonGround
- Bates-stamped appendices
- Verification/authentication statement

**Example:**
```
Interrogatory #5: "State the number of times Respondent failed to appear 
for scheduled custody exchanges in the past six months."

Response: Two (2) times. See Exhibit A (Exchange Compliance Log) for 
details. Specifically:
- January 5, 2024 at 6:00 PM
- February 2, 2024 at 6:00 PM
```

**Page Count:** Varies (depends on discovery scope)

## 2. CLIENT COMMUNICATION REPORTS

These are simplified, parent-friendly reports to share with clients (not for court filing).

---

### 2.1 Client Case Status Report

**File Name:** `Case_Status_Update_[ClientName]_[Date].pdf`

**Purpose:** Update client on case health and compliance status in plain English.

**When to Use:**
- Quarterly check-in with client
- Before mediation or hearing (prep client)
- Response to client asking "How is my case going?"
- Showing client areas needing improvement

**Contents:**
- "Your Co-Parenting Scorecard" (plain English)
- **Strengths:** What you're doing well
- **Areas to Improve:** What needs work
- **Your Ex's Compliance:** What they're doing well / not well
- **Recommendations:** Specific actions to improve compliance
- **Next Steps:** Upcoming deadlines, hearings, meetings

**Tone:** 
- Conversational, not legal jargon
- Encouraging where compliance is good
- Constructive where improvement needed
- Action-oriented (what client can do differently)

**Example Section:**
```
YOUR EXCHANGE COMPLIANCE: Excellent! ⭐

You've been on-time for 27 out of 28 exchanges (96%). This is really strong 
and shows the judge you're committed to the parenting schedule. Keep it up!

The one late exchange (Jan 19) was only 15 minutes and you communicated 
about traffic. That's the right way to handle unavoidable delays.

THEIR EXCHANGE COMPLIANCE: Needs Improvement ⚠️

Unfortunately, Mike has missed 2 exchanges and been late 3 times. This is 
something we can bring up at the hearing if his pattern continues.

RECOMMENDATION: 
Continue your excellent attendance. Document any future missed exchanges 
by the other parent (take screenshots of your GPS check-in and any messages).
```

**Page Count:** 3-5 pages (intentionally short)

---

### 2.2 Client Preparation Report (Hearing Prep)

**File Name:** `Hearing_Preparation_[ClientName]_[Hearing Date].pdf`

**Purpose:** Help client prepare for upcoming hearing by showing them the evidence.

**When to Use:**
- 1-2 weeks before hearing
- Before mediation session
- Client needs to understand case strengths/weaknesses

**Contents:**
- **What the Hearing Is About** (plain English)
- **Your Compliance Summary** (what you'll testify to)
- **Their Compliance Summary** (what they'll have to answer for)
- **Key Evidence** (what the judge will see)
- **Likely Questions** (what you'll be asked)
- **How to Testify** (tips for court)
- **What to Bring** (documents, evidence)

**Example:**
```
WHAT THE JUDGE WILL ASK YOU:

1. "Ms. Martinez, have you been on time to exchanges?"
   → YES. You've been on time 96% (27 of 28 exchanges). The one time you 
      were late, you called ahead. The evidence shows this clearly.

2. "Have you paid your share of medical expenses?"
   → YES. You've submitted all medical bills through ClearFund and they're 
      documented. Mike has disputed two bills but you have the receipts.

3. "How has communication been with Mr. Johnson?"
   → CHALLENGING. You've tried to keep communication focused on the kids, 
      but 14 messages from Mike were hostile and needed to be rewritten by 
      ARIA. We have examples.
```

**Page Count:** 5-8 pages

---

**File Name:** `Monthly_Summary_[ClientName]_[Month-Year].pdf`

**Purpose:** Automated monthly email to client showing case status (keeps attorney top-of-mind, provides value).

- Sent automatically 1st of each month
- Client doesn't have to request it

- **This Month's Compliance Score:** 85% (↑ up 3% from last month!)

  - Children's spring break schedule confirmed
- **Action Items for Next Month:**
  - None currently (you're doing great!)

**Tone:** Positive, brief, automated

**Page Count:** 1-2 pages (email-friendly)

## 3. INTERNAL CASE MANAGEMENT REPORTS

These are for attorney's own case preparation and strategy (not shared with client or court).

---

### 3.1 Case Health Dashboard Report

**File Name:** `Internal_Case_Health_[CaseName]_[Date].pdf`

**Purpose:** Attorney's own overview of case status for internal file review, case planning, or delegation.

**When to Use:**
- Preparing for hearing (attorney's notes)
- Assigning case to associate or paralegal
- Case review meeting with team
- Strategic planning

**Contents:**
- **Compliance snapshot** (all key metrics at a glance)
- **Risk assessment:**
  - Is this case escalating or improving?
  - What's the biggest risk factor?
  - Is emergency intervention needed?
- **Strategic notes:**
  - Strengths to emphasize
  - Weaknesses to address
  - Evidence gaps (what's missing?)
  - Next steps
- **Attorney's working notes** (can add annotations)

**Example:**
```
RISK ASSESSMENT: MEDIUM-HIGH ⚠️

Escalation Pattern: Yes - hostile messages increased 60% this quarter
Compliance Trend: Declining (73% down from 85%)
Biggest Risk: Parent B's pattern could justify custody modification
Emergency Risk: Low (no immediate safety concerns)

STRATEGY NOTES:
• Strong case for communication restrictions
• Evidence package ready for hearing
• Consider parenting coordinator recommendation
• Client needs coaching on documentation
```

**Page Count:** 2-4 pages

---

### 3.2 Evidence Gap Analysis Report

**File Name:** `Evidence_Gaps_[CaseName]_[Date].pdf`

**Purpose:** Identify what evidence is missing to strengthen case.

**When to Use:**
- Preparing for trial
- Discovery planning
- Identifying what to request from client
- Case weakness assessment

**Contents:**
- **Available Evidence:** What we have from CommonGround
- **Missing Evidence:** What would strengthen case
- **Evidence to Request:**
  - From client
  - From opposing party (via discovery)
  - From third parties (school, medical, etc.)
- **Timeline for Evidence Collection**

**Example:**
```
AVAILABLE: Strong exchange compliance data (GPS verified)
MISSING: School attendance records during each parent's time
ACTION: Request from client (form letter drafted)
DEADLINE: Due in discovery in 30 days

AVAILABLE: ARIA hostile message log
MISSING: Impact on children (therapist records)
ACTION: Subpoena child's therapist with court permission
DEADLINE: Need court order first
```

**Page Count:** 2-3 pages

---

### 3.3 Comparative Analysis Report (Multi-Period)

**File Name:** `Trend_Analysis_[CaseName]_[Date].pdf`

**Purpose:** Compare compliance across multiple time periods to show improving/declining patterns.

**When to Use:**
- Showing pattern over time (not just snapshot)
- Demonstrating that previous court intervention worked (or didn't)
- Modification requests require showing "substantial change"
- Long-term case monitoring

**Contents:**
- **Period Comparison:**
  ```
  Q4 2023: 85% compliance
  Q1 2024: 79% compliance (↓ 6%)
  Q2 2024: 73% compliance (↓ 6%)
  
  Trend: Consistent decline over 6 months
  ```
- **What Changed?**
  - Specific behaviors that got worse
  - Specific behaviors that improved
  - External factors (new relationship, job change, move, etc.)
- **Intervention Effectiveness:**
  - What has court tried? (warnings, make-up time, contempt)
  - Did it help? (data-driven answer)
  - What should be tried next?

**Page Count:** 5-8 pages

---

## 5. REPORT FORMATS & TECHNICAL SPECIFICATIONS

### 5.1 File Formats Available

**PDF (Primary):**
- SHA-256 verified
- Password protection optional
- Bates numbering optional
- Watermark optional ("DRAFT" or "CONFIDENTIAL")

**Word/DOCX (Secondary):**
- Editable for attorney notes
- Convert to PDF for filing
- Not court-ready until converted

**Excel/CSV (Data Export):**
- Raw data only
- For attorney's own analysis
- Can import into other tools

---

### 5.2 Report Security Features

**SHA-256 Verification:**
- Every court report includes hash
- Verify report hasn't been altered
- Court can verify authenticity via CommonGround

**Bates Stamping:**
- Optional for discovery
- CG-001, CG-002, CG-003... format
- Tracks document production

**Password Protection:**
- Optional on all reports
- Attorney sets password before download
- Prevents unauthorized viewing

**Watermarks:**
- "DRAFT" - for internal review
- "CONFIDENTIAL - ATTORNEY WORK PRODUCT"
- "FILED WITH COURT [DATE]" - after filing

---

### 5.3 Report Generation Speed

- **Simple Reports** (Exchange/Financial only): 30 seconds
- **Full Compliance Report**: 1-2 minutes
- **Emergency Report**: <5 minutes (priority processing)
- **Multi-Period Trend Analysis**: 2-3 minutes (more data to process)

---

### 5.4 Report Customization Options

**Date Range Selection:**
- Last 30 days
- Last 90 days (1 quarter)
- Last 6 months
- Last 12 months
- Custom date range
- Since court order date

**Include/Exclude Sections:**
- ☑ Exchanges
- ☑ Support
- ☑ Communication
- ☐ Holiday schedule (if none in period)
- ☑ ARIA interventions
- ☐ Raw data appendices (for shorter report)

**Detail Level:**
- Executive summary only
- Standard (recommended)
- Detailed (include all evidence)
- Ultra-detailed (include all raw data)

**Privacy Redaction:**
- Redact children's names
- Redact addresses
- Redact phone numbers
- Redact specific messages (for GAL, evaluator)

## 6. REPORT USAGE GUIDE

### When to Use Which Report

**Preparing for Court Hearing:**
→ **Full Compliance Report** (most comprehensive)

- Attorney can review before sending

**Quarterly Compliance Reports:**
- Auto-generate 1st of quarter
- Saved to attorney's dashboard
- Attorney reviews and files when needed

**Pre-Hearing Reports:**
- System detects court date in calendar
- Generates report 2 weeks before
